# Git 操作指南

## 基本 Git 指令速查表

### 初始化與狀態查詢

#### 1. git init
- **功能**：在這個資料夾開一個「新的 Git 時空」
- **使用時機**：第一次要在這個專案開始用 git 時
- **常見例子**：新專案第一件事就打這個

#### 2. git status
- **功能**：問 git：現在資料夾裡有什麼東西變髒了？
- **使用時機**：隨時想知道目前狀態（超常用！）
- **常見例子**：幾乎每做一步都先看一下 status

### 檔案管理

#### 3. git add [檔案名稱]
- **功能**：把這個檔案/資料夾「放進準備區（暫存區）」
- **使用時機**：決定要把哪些變更帶去下一個 commit
- **常見例子**：`git add index.html`

#### 4. git add .
- **功能**：把「所有有變更的檔案」全部放進準備區
- **使用時機**：懶人/小專案常用，改很多檔一次想全部提交就打這個
- **常見例子**：一次性提交所有變更

#### 5. git commit -m "訊息"
- **功能**：把準備區的東西「真的存下來」，並附上一張便利貼說明
- **使用時機**：覺得這次的修改可以定一個版本了
- **常見例子**：`git commit -m "完成首頁設計"`

### 歷史查詢與比較

#### 6. git log / git log --oneline
- **功能**：看目前存了哪些版本（歷史紀錄）
- **使用時機**：想知道之前都做了什麼
- **常見例子**：`--oneline` 比較好看

#### 7. git diff
- **功能**：秀出「還沒 add 進去」的修改差異
- **使用時機**：提交前想再確認一下改了什麼
- **常見例子**：常跟 status 一起用

### 檔案還原

#### 8. git restore [檔案名稱]
- **功能**：把這個檔案「還原」成上一次 commit 的樣子
- **使用時機**：不小心改壞了，想丟掉目前修改
- **常見例子**：`git restore style.css`

### 分支管理

#### 9. git branch
- **功能**：看目前有哪些分支，我在哪一個分支
- **使用時機**：想知道現在身處哪條時間線
- **常見例子**：看右邊有沒有 * 號

#### 10. git checkout -b [分支名稱]
- **功能**：開一條新分支 + 立刻跳過去
- **使用時機**：要開始做新功能、不想直接改壞主線
- **常見例子**：`git checkout -b feature/login-page`

### 遠端同步

#### 11. git push origin main
- **功能**：把本機的 main 分支推到遠端（通常是 GitHub）
- **使用時機**：想把程式備份到雲端/給別人看
- **常見例子**：記得第一次要先設定 remote

#### 12. git pull
- **功能**：把遠端的最新版本抓回來，跟本機合併
- **使用時機**：開工前、別人推東西後、要開始工作前
- **常見例子**：每天上班第一件事通常先 pull

## 常用工作流程

### 日常開發流程
1. `git status` - 檢查目前狀態
2. `git add .` - 將變更加入暫存區
3. `git commit -m "描述訊息"` - 提交變更
4. `git push origin main` - 推送到遠端

### 新功能開發流程
1. `git pull` - 先同步最新版本
2. `git checkout -b feature/新功能名稱` - 建立新分支
3. 進行開發...
4. `git add .` 和 `git commit -m "訊息"` - 提交變更
5. `git push origin feature/新功能名稱` - 推送分支

### 每日開工流程
1. `git status` - 檢查狀態
2. `git pull` - 同步遠端最新版本
3. 開始工作...

## 注意事項

- 提交前記得先用 `git status` 和 `git diff` 檢查變更
- 提交訊息要寫得清楚明瞭
- 重要功能開發建議使用分支
- 定期推送到遠端備份
