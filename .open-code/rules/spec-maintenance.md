# 規格文件維護規範

當進行代碼邏輯變更時，必須同步更新規格文件（requirements.md, design.md, ai-todo.md）。

## 1. 更新觸發條件
- 新增/修改功能、參數或業務邏輯。
- 性能優化或加入緩存機制。
- 修正邏輯、計算或條件判斷錯誤。

## 2. 文件職責分配
- **requirements.md**: 記錄「做什麼」 (What)。更新需求、驗收標準與使用者說明。
- **design.md**: 記錄「怎麼做」 (How)。更新架構、函數設計、資料模型與演算法。
- **ai-todo.md**: 追蹤進度。標記完成狀態、新增子任務。

## 3. 同步流程 (sync spec)
1. 識別變更影響範圍。
2. 按照 `Requirements -> Design -> Todo` 的順序進行更新。
3. 確保三個文件之間的術語與編號一致。
4. 提交修改前，進行品質檢查清單核對。

## 4. 品質檢查
- [ ] 需求是否都有對應的驗收標準？
- [ ] 設計是否與實際代碼實作一致？
- [ ] 任務狀態是否準確反映進度？
- [ ] 術語是否統一？
