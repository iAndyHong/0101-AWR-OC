# Andy 的 OpenCode 開發規範

## 第一部分：通用開發規範 (General Development Rules)

### 1. 核心原則 (Core Principles)
- **語言絕對禁令 (ZERO TOLERANCE)**：
  1. 嚴禁在思考過程與註解中出現任何英文單字或技術術語（代碼內建變數名、路徑除外）。
  2. 即使是內部推理邏輯，也必須百分之百轉化為繁體中文思維。若違反此項，視為任務失敗，必須重頭執行。
- **效率第一**：回答簡潔優雅，直指重點。
- **品質至上**：代碼需遵循最佳實踐，零錯誤零警告。
- **演算法保護 (CRITICAL)**：嚴禁未經授權擅自修改或「優化」核心交易演算法（如網格間距、手數縮放、對沖平倉邏輯）。

### 2. 溝通與協作
- **稱呼**：請稱呼我為 Andy。
- **用語**：一律使用繁體中文 (台灣用語)，嚴禁使用中國用語。
- **完成提示**：任務結束後輸出 "完成，待命中..."。

### 3. 標準工作流程
1. **任務啟動**：必須執行 `say -r 220` 提醒 Andy。語音內容可根據任務性質具備彈性（例如："開始修正邏輯"、"開始編譯驗證"）。
2. **實作與驗證**：遵循零警告政策執行開發與編譯。
3. **任務完成**：必須執行 `say -r 220` 報告狀態。語音內容應清楚說明結果（例如："修正完成，編譯成功"、"邏輯已更新，待命中"）。
