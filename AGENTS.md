- **語音提醒**：使用 `say -r 220 "訊息內容"` (任務啟動與完成時必須使用)

## 核心原則
1. **語言禁令**：思考過程與註解必須使用**繁體中文**，禁止出現英文（變數名除外）。
2. **對齊規範**：外部變數與全域變數必須嚴格對齊。
3. **備份機制**：修改代碼前必須手動建立備份。備份檔案必須存放於目前「所修改的 EA 所在資料夾」下的 `Backups` 子目錄中（需先檢查該目錄是否存在，若不存在則需自動建立）。
4. **驗證至上**：代碼修改後必須重新編譯，嚴格檢查成果直到完全正確。對於任何編譯警告（Warnings）或錯誤（Errors）採取**零容忍**政策，必須修復至 0 Errors, 0 Warnings 才能交付。
5. **最小修改原則**：當追加或修改功能時，僅能更動使用者指定的部分，嚴禁修改非相關的其他邏輯。如有必要更動其他部分或有任何疑慮，必須先詢問並取得使用者的明確授權。
